# Space RTS – Module-based Station Builder

## О проекте
Проект представляет собой RTS с элементами Factorio, где игрок строит космическую станцию из модульных блоков. Станция находится в центре карты и подвергается нападению врагов, приходящих с краёв карты. Помимо станции, доступны капитальные корабли (меньше станции, но могут перемещаться по карте) и, возможно, истребители (с фиксированными пресетами модулей). Основная механика строительства основана на сетке ячеек, в которые устанавливаются модули (двигатели, орудия, броня, щиты). Модули определяют функциональность корабля.

## Реализованные системы

### 1. Камера (RTSCameraController)
- Орбитальное вращение (yaw) вокруг целевой точки (Q/E или средняя кнопка мыши)
- Наклон (pitch), зависящий от зума (кривая `zoomToPitchCurve`)
- Панорамирование (WASD, края экрана)
- Зум (скролл) с плавной интерполяцией
- Скорость движения зависит от зума (чем ближе к земле, тем медленнее)
- Привязка к круглым границам карты (MapBounds)
- Глобальные настройки LOD передаются в ShipController

### 2. Процедурная генерация подложки (ProceduralFoundation)
- Создаёт основу корабля по набору ячеек (HashSet<Vector2Int>)
- Верхняя грань объединяет соседние ячейки в прямоугольники (greedy meshing) для оптимизации
- Боковые грани генерируются только там, где нет соседней ячейки
- Нижняя грань – простые квадраты (возможно улучшение)

### 3. Система модулей (ShipController)
- Каждая ячейка может содержать модуль определённого типа (ядро, оружие, двигатель, броня)
- Модули – простые кубы с цветовой маркировкой
- Поддержка сохранения/загрузки (бинарная сериализация, данные в ShipData)
- Стандартная станция создаётся из пресета (StationPreset)

### 4. LOD (уровни детализации) с биллбордом
- При удалении камеры на расстояние > `distanceToSwitch` 3D-модель отключается, вместо неё показывается Canvas с текстурой-импостером
- Текстура генерируется один раз при спавне корабля (ортографическая камера, питч -90° для вида сверху)
- Импостер имеет фиксированный размер на экране (64x64 пикселя) и всегда повёрнут к камере (Billboard)
- Внутри Canvas изображение дополнительно поворачивается так, чтобы нос корабля всегда указывал на «вверх» камеры – игрок видит ориентацию

### 5. Границы карты (MapBounds)
- Круглая карта с заданным радиусом и центром
- Ограничивает движение камеры и кораблей
- Визуализация границ (тор) в режиме редактора и через MapBoundsVisualizer

### 6. Выделение объектов (RTSActorParent + ObjectSelector)
- Базовый класс для всех выделяемых объектов
- Клик левой кнопкой мыши выделяет/снимает выделение

## Настройка сцены

1. **Камера**  
   - Создайте пустой объект или используйте Main Camera.  
   - Добавьте компонент `RTSCameraController`.  
   - В инспекторе **обязательно** установите `Impostor Pitch Angle = -90`. Это необходимо для правильного захвата текстуры импостера (вид сверху).

2. **Границы карты**  
   - Создайте пустой объект, добавьте компонент `MapBounds`.  
   - Установите `Map Radius` (например, 50) и при необходимости `Map Center` (по умолчанию (0,0,0)).  
   - Для визуализации границ добавьте на тот же объект компонент `MapBoundsVisualizer` (требует материал).

3. **Менеджер игры**  
   - Создайте пустой объект, добавьте компонент `GameManager`.  
   - Перетащите в поле `Foundation Material` любой подходящий материал (например, стандартный `Default-Material` или свой). Этот материал будет использоваться для подложки всех кораблей.  
   - При необходимости измените имя файла сохранения.

4. **Дополнительно**  
   - Убедитесь, что на сцене есть направленный свет (Directional Light) для освещения.  
   - Камера должна иметь тег `MainCamera`.  
   - Для отладки LOD при нажатии F12 в консоль выводится информация о зуме, питче и состоянии всех кораблей.

## Планы на будущее

- **Строительство**  
  - UI инвентаря, выбор модуля, установка на свободную ячейку (с проверкой соединения с ядром).  
  - Расширение сетки (кнопка «Expand»).  

- **Бой**  
  - Система повреждений (отключение модулей, уничтожение ядра).  
  - ИИ врагов, спавн за границами карты.  
  - Движение кораблей (2D физика).
 
- **Концепция повреждений и ядра**
  - **Ядро** – особый модуль, который нельзя снять или переместить. Уничтожение ядра приводит к мгновенной гибели корабля (удаление объекта).  
  - **Остальные модули** (оружие, двигатели, броня, щиты) имеют собственное здоровье. При получении урона модуль отключается (меняет цвет/свечение) и перестаёт выполнять свои функции, но остаётся на месте.  
  - **Ремонт** – любой модуль, кроме ядра, можно починить (например, с помощью ремонтных дронов или специального модуля), восстановив его здоровье и функциональность.  
  - **Связность** – жёсткой проверки соединения с ядром не требуется, так как все модули устанавливаются на подложку (основу), которая уже обеспечивает физическую связь. Хп основы обеспечивается ядром.
  - **Визуализация повреждений** – планируется изменять материал/цвет модуля в зависимости от оставшегося здоровья (зелёный → жёлтый → красный) и добавлять эффекты искрения при поломке.

- **Капитальные корабли**  
  - Аналогичны станции, но с возможностью движения.  
  - Меньше слотов, но больше мобильность.  

- **Истребители**  
  - Производятся по пресетам модулей, не подлежат изменению после создания.  

- **Оптимизация**  
  - Кэширование текстур импостера (ImpostorCache).  
  - LOD для врагов.  

## Текущие известные ограничения
- Подложка генерируется только для прямоугольных/произвольных форм, но без учёта внутренних пустот (сейчас это не нужно).  
- Модули пока только визуальны, логика не реализована.  
- Сохранение/загрузка работает, но не поддерживает динамические изменения формы (будет добавлено).
